## 0. 一句话结论(先读,可能不是你期待的)

**跨事件对冲在数学上能做、引擎复用度很高,但流动性现实决定了它大多数时候会诚实地返回 PARTIAL/NO_GO——只有"进决赛 NO"这一个工具、且只对前 ~6 强球队、能填进 $500–1500 的仓位。**

所以这个扩展的 MVP 价值是**完整性与诚实**("我们连跨事件也查了,这一个过线、其余为什么不过线"),**不是**"解锁一大堆新的 GO 对冲"。如果你期待的是后者,数据说现在做不到(临近开赛流动性变深前)。

**建议:做,但收窄到一个工具,且第一件事是修深度闸门(否则会骗用户)。** 详见 §5。

## 1. 市场与流动性现实(实盘核验)

针对持仓"西班牙夺冠",能在西班牙输时赔付的跨事件市场(对冲腿 = 买该市场的 **Spain NO**):

| 对冲 | 事件 slug | 结构 | Spain NO 价 | 近触可买(+2c / +5c) | 可用? |
|---|---|---|---|---|---|
| **进决赛 NO** | `world-cup-nation-to-reach-final` | 每队二元 | 0.725 | **$1,555 / $2,778** | ✅ 唯一真能填的 |
| 进四强 NO | `world-cup-nation-to-reach-semifinals` | 每队二元 | 0.59 | $268 / $276 | ❌ 太薄 |
| 进八强 NO | `world-cup-nation-to-reach-quarterfinals` | 每队二元 | 0.42 | $21 / $673 | ❌ 太薄(深处是假墙) |
| 小组出线 NO | `world-cup-team-to-advance-to-knockout-stages` | 每队二元 | 0.0285 | $204 | ❌ 价≈0.03,无对冲价值 |
| 西班牙淘汰轮次 | `world-cup-spain-stage-of-elimination` | negRisk 8 档 | — | 每档低四位数 | ❌ 太薄,且有退化档 |

**关键陷阱(复查确认):** 进八强 NO 的"总挂单 $312k"、出线 NO 的"$6.1M"都是**贴近 $1 的假墙**,近触真实可填只有 $40–$300。现有的 `notionalDepth(asks)`(整边求和)**看不出来**——这是最大风险,见 §5。

**对比尺度:** 夺冠市场 ~$2.34B 成交 / ~$504M 流动性;最好的跨事件盘(进决赛)全部挂单合计才 ~$15.7k。跨事件长尾比它要对冲的仓位**薄 3–4 个数量级**。

**前 ~6 强对称性成立(仅进决赛 NO):** 法国 +2c≈$1,607、英格兰≈$1,105、巴西≈$976。前 4–6 队能撑 $500–1500 的对冲,再往下迅速变薄。

**好对冲 vs 坏对冲:**
- ✅ 好(西班牙被淘汰时赔付,买 NO):进决赛/四强/八强 NO、出线 NO。但只有进决赛 NO 够深。
- ❌ 坏:小组第一 YES、进 X 轮 YES(与"夺冠"正相关,加风险);小组第一 NO(西班牙小组第二仍可夺冠,不干净)。
- **嵌套关系:** {出线}⊂{进八强}⊂{进四强}⊂{进决赛}⊂{夺冠}。**叠加多个 = 重复计同一批淘汰结果**,引擎必须当成**一条嵌套阶梯**,不是独立腿。

## 2. 核心建模难题与方案

**难题:** 现在引擎只建模一个分区——"60 队谁夺冠"。每条腿在这个分区的一个子集里赔付。但"西班牙进决赛"**不是**"谁夺冠"的确定性函数(巴西夺冠并不告诉你西班牙有没有进决赛)。所以分区不够用了。

**方案:结构化蒙特卡洛(不是扩展解析分区)。** 理由:扩展分区会被迫给每个 (冠军, 西班牙轮次) 格子指定联合概率,而你只有边际(阶梯市场给西班牙边际、夺冠市场给冠军边际),硬拼会引入"假设的相关性"——正是本项目哲学反对的。MC 只用你能从价格读到的**边际** + 一条结构约束(`冠军=持仓队 ⟺ 轮次=夺冠`),其余由模拟自然填充,从不拟合 ρ。

**MC 怎么work:** 每次模拟产出一个 `SimDraw = { champion, rounds: {team→furthestRound} }`。每条腿的赔付变成一个**谓词** `paysInSim(draw)=>boolean`(同事件腿只看 champion;跨事件腿读 `rounds[team]`)。在**同一个 draw** 上评估所有腿 → 自动一致 → 同事件和跨事件选项能放进**一个引擎、一个 P&L 分布、一个菜单**。

**生成器(关键):**
1. 把西班牙的阶梯市场价去 vig + 用 isotonic(PAVA)强制单调(`P(夺冠)≤P(进决赛)≤…≤P(出线)`),转成"最远轮次"的 PMF。
2. **把夺冠档钉死在深的夺冠市场**(`pmf[6]=q_devig[Spain]`),其余重整到 `1-q`。这是冠军轴与阶梯轴的唯一联结。
3. 抽样:先从 `q` 抽冠军;若某队=冠军→轮次=6;否则从"非冠军条件分布"抽轮次。这样**冠军边际和阶梯边际都按构造精确**,无拟合。
4. 用种子 PRNG(`mulberry32`,种子来自数据快照)→ **同快照 = 同裁决**,可测、不闪烁。

只有持仓队 + 出现在某条对冲腿里的队需要阶梯(通常就 1 队),其余只需冠军抽样 → MC 很便宜。

**验证结论(已数值复现):** 生成器数学正确——冠军与阶梯联合一致、单调嵌套被强制、夺冠态无重复计、maxLoss/CVaR 正确产出。

## 3. 引擎改动(全部是叠加,复用度高)

原则不变:**一个费率函数、一个裁决。** MC 只改"P&L 分布怎么算",不改"谁来裁决"。

| 模块 | 改动 | 性质 |
|---|---|---|
| `lib/types.ts` | 加 `Round`、`SimDraw`、`LegPayoff` 联合类型、`CorrRule += "CROSS_EVENT"` | 纯叠加 |
| `lib/sim/generator.ts` | **新**:`buildLadderPmf` / `simulateOnce` / `simulate` + 种子 PRNG + PAVA + 插值 | 纯函数,可单测 |
| `lib/netcost/benefit.ts` | 加 `pnlDistributionSim(draws, SimLeg[])`;`riskMetrics` **不动**(它已经吃任意权重的 `PnLPoint[]`,每个 sim 权重 1/N) | 叠加 |
| `lib/sizing/strategy.ts` | 加 `expectedLogWealthSim`(把"60 outcomes 按 q"换成"N sims 按 1/N");band/haircut/capacity 逻辑不动 | 叠加内核 |
| `lib/sizing/decide.ts` | 泛化输入接受 `draws + paysInSim[]`,分支两处分布构造;**裁决算术不变**;但**文案要改**(见 §5 issue 2) | 小逻辑改动 |
| `lib/correlation` | ρ 闭式不再参与风险数学(MC 经验捕获),**保留作解释**(`crossEventEdge` 的 why);`devig` 仍在关键路径 | 角色从"输入"变"叙述" |
| `lib/pipeline.ts` | 发现西班牙阶梯市场 → 建 `pmf` → `simulate()` 一次 → 加 `reach-final-no` 策略(`paysInSim`)→ 走**现有** `evaluateStrategy`/`decideStrategy`/排序 | 接线 |

**优雅的退化性质:** 若某队没有任何阶梯市场,`pmf` 退化为 {夺冠 q, 不夺冠 1-q},MC 等价于今天的分区引擎 → **零回归**。这也是为什么 Phase 1 能先于真实数据落地并用回归测试守住。

## 4. 发现机制(从持仓找阶梯市场)

1. **从夺冠事件的 event 级 tag 透视**:`gamma /events?tag=<wc2026>` 枚举所有 WC 事件。
2. **按 `groupItemTitle`/question 匹配持仓队**:复用现有 `tokenSetScore`/`teamQuery` 模糊匹配 + `WC2026_TEAMS` 别名。
3. **按问题文案分类到轮次**("advance"→1,"reach the final"→5,...),小正则表;匹配不上就丢弃,不猜。
4. **纳入闸门**:轮次明确 + **NO 盘过深度闸(见 §5)** + 未结算。夺冠档永远取自深的夺冠市场。
5. 发现服务两用途:(i) 取价建 `pmf`(所有存在的档),(ii) 取可交易对冲腿(上层档的 NO 边,过深度的)。

## 5. 必须先修/做对的几件事(开工前要认的账)

### 🔴 HIGH 1 — 深度闸门:这是头号风险,第一件事
现有 `pipeline.ts:251` 用 `notionalDepth(book.asks)`(整边求和)来定 `maxScaleUsd`。在进八强 NO 上这是 **$312k**、出线 NO 是 **$6.1M**——全是贴近 $1 的假墙。直接复用 = 在根本填不进去的盘上推 GO,让 retail 下单后吃 5–15c 滑点。**必须改成"近触可填(walk 到 best ask + Nc 价格上限)"的深度度量**,这是一个**新 helper**,不是复用。**这一条做对,产品诚实;复用旧的,产品就在新增的市场上撒谎。**

### 🔴 HIGH 2 — 裁决文案要分策略类型
`decide.ts:204–207` 的 NO_GO 文案("如果一支不在这个对冲里的队伍夺冠,你会同时输掉持仓和这笔注,这是 side bet")是**专门针对同事件对手注**的,对"进决赛 NO"这种**真能降最坏损失**的跨事件腿是**错的**。需要给 `Strategy` 加一个 `crossEvent`/kind 标志,文案随类型变——这是一处**真实的小逻辑改动**,设计里"裁决逻辑不动"的说法在文案层面不准确。

### 🔴 HIGH 3 — 阶梯去 vig 来自薄盘,会扭曲尾部
建 `pmf` 的每一档用的是那些 3–4 OOM 更薄、价差更宽的盘 → 烂价 → 扭曲的轮次分布 → 扭曲的跨事件 CVaR。MC 的"采样噪声"caveat 盖不住这个"价格质量噪声"(后者才是主误差)。修:(a) `uncertaintyHaircut` 随档位盘的价差/深度放大;(b) 档位盘太薄就**剔除该档、插值**;(c) 在 `facts` 里**强制披露**"轮次概率是从薄盘推断的"。

### 🟡 MED — 还要处理
- **西班牙淘汰轮次事件有个退化"Other"档(0.5/0.5、流动性 $0)**:去 vig 前必须剔除零流动性占位档,否则 overround 被污染。
- **跨对手阶梯对冲(如"巴西进决赛 NO")先不做**:同半区两队不可能都进决赛,但我们独立抽轮次 → 偏差**方向是错的**(低估对冲价值),不只是不精确。等 bracket 种子(`wc2026-structure.ts` 的 groups/half,目前 TODO)再做。
- **sizing sim 内核性能**:N≈20k sims × 网格搜索(200+40 步)× 多策略。要**预计算每 sim 的 held-win 布尔 + 每腿 `paysInSim` 位集**,避免重复评估。

### 🟢 LOW — 测试纪律
- **回归测试先写**:空阶梯 MC 必须复现今天分区引擎的 CVaR/maxLoss/裁决(在 MC 容差内)——这是"零回归"的keystone,先于任何 sim 代码碰共享路径。
- **裁决稳定性测试**:进决赛 NO 的 η 很可能贴着 3.0 阈值,MC 噪声可能让 GO/PARTIAL 翻转。必须有硬验收:**选定 N 下,10 个种子裁决不翻转**。

## 6. 范围(批准的就是这个)

**做(MVP,收窄):**
1. **近触可填深度闸门**(walk 到价格上限,非 `notionalDepth`)—— 一切的前提,先做。
2. **只做"进决赛 NO",只对前 ~6 强**;硬闸:近触可填 < 仓位 → PARTIAL/NO_GO;否则报"无可填的跨事件对冲"。
3. `lib/sim/generator.ts`(纯、种子化)+ `pnlDistributionSim` + `expectedLogWealthSim`(位集预计算)。
4. **回归测试先行** + 裁决稳定性测试。
5. 裁决文案分策略类型(跨事件不再说"side bet")。
6. 发现:剔除零流动性占位档;档位盘质量 → `uncertaintyHaircut`。

**明确推迟:**
- 进四强/八强/出线 NO(填不进去)、西班牙淘汰轮次作对冲(薄+退化档)。
- **跨对手阶梯对冲**(偏差方向错,等 bracket 种子)。
- 大洲/超集篮子对冲。
- 逐场比赛模拟(不必要)。

| 阶段 | 内容 | 产出 |
