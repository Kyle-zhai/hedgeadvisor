## 0. 一句话结论(和你预想的可能相反——好消息和坏消息各一个)

**好消息:引擎做数学的地方早就与领域无关了。** 每个数值模块(netcost、correlation、sizing、decideStrategy 裁决、排序)只吃抽象(`Outcome{label,q}`、`PnLLeg{shares,paysIn}`、`FeeSchedule`),**零足球**。"补集对冲(买自家 NO)"对"Newsom 赢 2028"和"西班牙夺冠"是**同一套数学,零新建模**。那条核心定理("同一互斥事件内唯一真对冲是补集,对手篮子反而抬高最坏损失")是关于**互斥分区**的,不是关于足球的,对选举/加密分区**逐字成立**。

**坏消息:真正的活不是数学,是"解析器(resolver)"。** 复查实盘发现:对最关键的查询 **"Newsom wins 2028",现有方案的通用搜索返回空**(只搜到一个已关闭的 2024 普选事件,活的 `presidential-election-winner-2028` 根本不出现)→ `not_found`。Polymarket 自由文本搜索召回差、偏向过期市场。**引擎再领域无关,解析器映射不对就没有可演示的东西。**

**建议:做,但 MVP 的重心是硬化解析器 + 只上线"美国政治"这个滩头堡,并把跨事件 LLM 分类器换成一张静态表(把假精度风险挡在 MVP 之外)。**

## 1. 领域就绪度(实盘 + 独立复查,按"近触可填 NO 深度"实测)

"近触可填 NO" = 从 best ask 走到 +3c 时,favorite 的 NO 这边能买进多少美元(持有 favorite 的人要买的补集对冲)。**只看整边求和会被假墙骗**(如 Newsom-NO 整边 $3.57M,近触只有 $28.7K)。

|---|---|---|---|---|
| **美国政治** | `presidential-election-winner-2028` | ✅ ~34 | Newsom-NO **$28.7K** / Rubio $53K / Vance $125K | ✅ 够 10× 以上 |
| **美国政治** | `democratic-presidential-nominee-2028` | ✅ ~34 | Newsom-NO **$23.8K**(AOC $103K, Ossoff $221K) | ✅ |
| 宏观/美联储 | `fed-decision-in-june` | ✅ 5 档 | No-change-NO best ask **0.006**(near-worthless) | ⚠️ 结构完美,经济上≈0(除非有争议会议) |
| 宏观/美联储 | `how-many-fed-cuts-2026` | ✅ 13 档阶梯 | "0 cuts"-NO **$7.5K** | ⚠️ 偏薄 |
| 体育(非足球) | `f1-drivers-champion` | ✅ | Antonelli-NO **$1.9K**(favorite 近 50/50,NO 贵且少)/ Russell $7.2K | ⚠️ favorite 那条最薄 |

**排序(真能撑一个有把握的 $500–2000 补集对冲):**
1. **美国政治 —— 明确就绪。** 唯一一个 **favorite 的 NO 都有五位数近触深度**的领域,全年(2028 周期 + 各州 + 外国选举 negRisk $37–102M)。**这一个就把赛后断崖补上了。**
2. **F1 / MLB —— 有条件可用,但比想象薄。** 深度是反的:**favorite 的 NO 最薄**(favorite 价在 0.5–0.7,NO 贵且挂得少),有深度的是中游。真正的对冲者持有 favorite → 经常 PARTIAL。需要赛季日历。
3. **宏观/美联储 —— 结构完美、经济上≈0**(no-change 0.9945,补集 0.006 是彩票)。只在有争议会议有用,事件驱动。
4. **加密 —— 没就绪**(深的结构错,对的结构薄 + 1 天到期)。

## 2. 核心架构(复用度极高 —— 数学几乎不动)

### 2.1 去足球化(全是删改/参数化,无新数学,约 1 天)
| # | 位置 | 改动 |
|---|---|---|
| 1.1 | `pipeline.ts` `DEFAULT_SLUG` | 去掉 `"world-cup-winner"` 硬默认;eventSlug 来自通用解析(§2.2)。env 仅留作测试/演示可选 pin |
| 1.2 | `resolve.ts` `teamQuery()` | 改名 `outcomeQuery()`;足球停用词(win/cup/world/fifa)换成领域中性停用词 + 金额/日期归一("$150k by Dec"保住 `150k dec`) |
| 1.3–1.6 | comments + `decide.ts:206` + `structural.ts`/`types.ts` 文案 | "team"→"outcome";"if a **team** outside wins"→"if an outcome outside resolves YES";`why` 动词参数化("wins"→"resolves YES") |

### 2.2 通用解析器(★ 真正的活,之前严重低估)
现在解析是 **slug 优先**(你得先知道事件)。用户不知道 slug。需要两段式,但**单靠 `/public-search` 实测会对旗舰查询返回空**。所以:
1. **Stage A 找事件(硬化版,这是 MVP 的硬核):** 合并 `/public-search` **与** `/events?closed=false&order=volume24hr&tag_id=…`(实测可用);打分**硬偏向 `open && negRisk && volume24hr>阈值`** + 要求年份/实体 token 命中;把"没有活的结构化市场"当**一等结果**,不是边角。
2. **Stage B 找结果(复用现有 `resolvePosition` 主体):** 用 `tokenSetScore(outcomeQuery(query), groupItemTitle??question)` 在事件内匹配,阈值不变。
3. **事件级歧义**:`HedgeResponse.candidates` 加 `eventSlug` 字段("你是指 *2028 民主党提名* / *2028 总统胜者*?")。
4. **🔑 缺失的关键测试:** 一套 **~10–15 条真实查询的实盘解析测试**("Newsom wins 2028"、"Vance president 2028"、"BTC above 150k"…)。按现状这测试会**失败**——这正是为什么它必须在 MVP 关键路径上。

### 2.3 每事件费率(★ 唯一一处正确性关键的改动)
`resolve.ts:67` 给**所有**市场硬默认 `feeRate:0.03`(体育档)。加密短期市场是**不同档(那个 1.80% 档)**。费率字段已经按市场穿过 `MarketRef`,引擎就绪——只需解析时按 **tag + 期限(tenor)兜底**填对的档。**填错 → η 用错成本基准 → GO/NO_GO 翻错**。必须配 crypto-vs-sports 费率测试。

### 2.4 阶梯形事件的"对手篮子"守卫(🟡)
在价格桶事件(BTC $64k–66k)上,top-q"对手"是**相邻桶**,买相邻桶更像**跨式(straddle)不是对冲**。`pipeline.ts` 的 rival/basket 策略在**阶梯形 negRisk 事件上要抑制或改名**(`why` 文案的"team beats you"框架对桶无意义)。

## 3. 跨事件结构嵌套:换方案 —— 用静态表,不用 LLM(把假精度挡在 MVP 外)

原计划用 LLM 关系分类器(判 SUBSET/EXCLUSIVE/CAUSAL,只让逻辑关系进定量对冲)。**复查发现它的核心防线不够:**

- **价格单调性校验是必要不充分。** 真 `A⊆B` 必满足 `q_A≤q_B`,但**因果对也能轻松通过**:反例(实盘量级)"美联储 2026 降息≥1 次" q≈0.29 与 "BTC 年底 >$150k" q≈0.15 → 0.15≤0.29 ✓ 单调性过,但根本没有包含关系。若 LLM 误标 SUBSET,引擎就 `P(A∧B)=min=0.15` 造一个**假联合**、算出自信的 ρ、定出假对冲——正是产品要避免的。单调性只能**证伪**坏 subset,不能**确认**真 subset。
- **关键词筛justification 是可被绕过的表皮**——已决定标 SUBSET 的模型会把理由说得像干净包含。

**结论:MVP 砍掉通用 LLM 分类器,换成一张小小的"静态声明关系表"。**
- 只放**定义上必然包含**的关系:`提名 ⊂ 总统`(赢总统**定义上**需要先获提名)、`初选 → 大选`、`分区 ⊂ 联盟 ⊂ 总冠军`等——几十行,人工声明,可信。
- 每条仍用**价格单调性校验**(`q_A≤q_B`)做运行时 sanity check;不过就拒。
- 标 `provenance:"PRIOR"` + haircut + 披露。
- **零 LLM 信任风险,拿到 ~90% 的价值**(美国政治那个真实用例:`nominee⊂president`)。通用 LLM 分类器是**以后单独的赌注**,不进 MVP。

## 4. 头号风险

**不是数学,是解析器。** 整个方向 A 的卖点是"用大白话输入仓位 → 得到诚实对冲"。实测现有方案对旗舰查询 "Newsom wins 2028" **返回空/错事件**,加密整个不行。引擎领域无关是必要但不充分——Stage A 映射不可靠,就没有可演示的产品。原架构把它当"一个调用 + 一个解析器"轻描淡写,实际是 MVP 里**被低估的硬核**。

(次大风险,仅在 stretch 阶段:因果对被误标 SUBSET、过了单调性、产出假精度对冲 —— 已用"静态表替代 LLM"从根上避开。)

## 5. 范围(批准的就是这个)

**做(MVP):**
1. **硬化的通用解析器**(真正的活):合并 search + tag/volume 列举;硬偏 `open&&negRisk&&volume` + 年份/实体命中;"无活的结构化市场"一等结果;事件级歧义类型;**~10–15 条实盘解析测试(缺失的 keystone)**。
2. **去足球化引擎**(§2.1,机械改名/文案,保持 WC 黄金回归测试绿)。
3. **每事件费率(tag + 期限兜底)** + crypto-vs-sports 费率测试(唯一正确性关键改动)。
4. **只上线"美国政治"** 这个验证过的滩头堡;F1/MLB 用**流动性闸**(favorite 近触 NO ≥ 仓位 × 余量)门控,够不到就让 `decideStrategy` 诚实返回 PARTIAL(它已经会)。
5. **阶梯形守卫**:价格桶事件上抑制/改名对手篮子策略。
6. **静态声明关系表**(`nominee⊂president` 等)+ 单调性校验 + `PRIOR` 标注,覆盖那一个真实的跨事件用例。

**明确推迟:**
- **通用 LLM 跨事件分类器**(假精度风险,用静态表替代)。
- 加密、休赛期体育、同域 MC 阶梯。
- 大洲/超集篮子。
- 因果/跨域关联(那是方向 B,本次明确不做)。
