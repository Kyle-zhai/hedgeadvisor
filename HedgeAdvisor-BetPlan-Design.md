## 0. 一句话结论(诚实,且有一个必须先认的安全问题)

**做,但 v1 收窄到:只用"胜平负"市场 + 滑块只放"平衡↔保守对冲"那一半;最激进那一端("全力表达观点")暂时锁住,因为它正好是财务上和道德上最危险的面,必须先补三块代码。**

**最危险的发现:现在的走簿函数没有价格上限。** 它会把整条挂单梯子吃完,即使填在触价的 3–12 倍也照样返回 `capacityHit:false`。实测"英格兰 0:3"这个比分:触价 0.024,但 $200 会一路吃到均价 0.085 / 最差 0.289,而引擎报告"成交正常"。而**滑块最激进那一端,按设计就是去找"最便宜、赔率最高的冷门腿"——正好一头扎进这个陷阱。** 这同时是用户最差的财务结果 + 最严重的诚实违背 + 产品定位的直接背叛。所以激进端必须先修。

好消息:架构对**保守↔平衡**那一半复用度极高(P&L 分布 + 走实盘簿 + de-vig + 唯一裁决都直接用),那一端就是你现在已经验证过的对冲产品。

## 1. 实盘市场现实(决定能下注什么)

每场对阵有一套"8 桶"市场(64 场实盘 fixture)。"近触可填"= 走到触价 +3c 内能买进的美元。

| 桶 | negRisk/分区? | 可填性($20–200) | v1 |
|---|---|---|---|
| **胜平负(1X2)** | ✅ 干净 3 选 1 分区(ΣYES≈1.005) | ✅ 极深($85k+) | **上** |
| more-markets(让分/大小球/BTTS 主线) | ❌ **不是分区**(negRisk=false,ΣYES=10.72,各腿重叠) | 主线深($12k–51k),非主线是假墙 | **推迟** |
| 精确比分 | ✅ 17 格分区(ΣYES≈1.095,**vig ~9.5%**) | favorite 格能填(1:0≈$83),**尾部是假墙**(0:3 真实只有 $8.90) | **仅作单腿加点,逐格闸控** |
| 半场结果 | ✅ 3 选 1 | 可用 | 推迟(phase 2) |
| first-to-score / 2H / 角球 / 球员 props | 多为 ❌ | 价差 4–94c,大量是死盘 | **拒绝** |
| 球队期货(夺冠/晋级/进决赛) | ✅ | 最深 | 已有(对冲腿) |

**关键纠正(复查抓出的)**:
- 精确比分是 **17 格不是 26**(26 是已结束的俱乐部世界杯版)。
- more-markets 让分/大小球**不能**放到"胜平负"3 格分区上——那需要拟合"赢球时的进球分布",正是产品禁止的拟合相关性。要用让分/大小球,必须以**精确比分 17 格**为骨架、按"比分是否落在集合内"做**确定性成员映射**(诚实),但尾部薄。
- 球员 props 中位价差 **0.94** —— 死盘,默认拒。

## 2. 发现/解析(★ 现在完全没建,是真实市场保证的命门)
- **`search` 参数是坏的**(搜 fifa 返回 Kraken IPO、Macron)。**必须用 tag 枚举**:`GET /events?closed=false&active=true&tag_slug=fifa-world-cup&limit=100&offset=N` → 592 事件/64 场。直接 `slug=` 查询可靠。
- **slug 不能从队名合成**:`fifwc-{home3}-{away3}-{YYYY-MM-DD}`,但队码不一致(kr vs kor、civ vs cvi、prt vs esp)。**必须从实盘 tag dump 建 `{队名→码}` 和 `{(码A,码B,日期)→fixture}` 映射,按日期+两队匹配**,不能字符串猜。
- ⚠️ **"西班牙 vs 葡萄牙"正确地不是真实赛程**(不同组)→ 引擎必须拒掉、并推荐真实对阵(如 `fifwc-ury-esp-2026-06-26`)。这正是你要的"只用真实在售内容"。
- **两阶段现实闸**(必须):① slug 存在+active+未结算(gamma);② **所选那条腿的盘有 ≥ 最小注额的近触可填深度**(CLOB 走簿)。现状 `resolvePosition` 只能按 slug 钉一个事件、模糊匹配队名——**对阵解析、队码图、parseBetIntent 全都没建**。

## 3. λ 滑块目标(修正后)
用户有观点 + 固定可调预算 B,在真实相关市场上分配。λ = 风险厌恶,在有效前沿上从"激进表达/最大上行"到"保守/最小方差/对冲"。

目标(均值-方差形式):`maximize  U(x) − λ·Var[P(x)]  s.t. Σcost(x) ≤ B, x ≥ 0`,P&L 分布复用 `pnlDistribution`。修正三点(复查抓出):
- ⚠️ **激进端的 U 写错会退化**:原设计 `U=Σ_{i∈观点} q_i·P_i` 实际会买**最高概率(接近 favorite)**的格子,正好和"激进=买冷门高赔"相反。改成 `U(x)=max_{i∈观点} P_i(x)`(最佳情形赔付)或 1/q 加权;**滑块交易的是方差,不是 q 加权**。两端要重新推导。
- ⚠️ **不是凸优化**(Var 是关于凹的份额向量的二次型)→ 是启发式网格搜索,小 M 没问题,但别号称凸/唯一。
- ⚠️ **λ 要按实盘 `Var_max` 归一化**(去掉全局魔法常数 κ),否则滑块"手感"会随观点腿价格(0.13 冷门 vs 0.57 热门)漂移。s=0.5 应恒为"最小方差↔最大方差分配的中点"。

## 4. 必须先补的代码(BLOCKER,照搬现有会出错/会骗用户/会劝赌)

### 🔴 1 — 带价格上限的走簿(头号,挡住一切)
现 `walkBookBuyBudget`(walk.ts:80–108)**无价格上限**,填到 3–12 倍触价仍 `capacityHit:false`。新增 `walkBookBuyBudgetCapped(book, budget, maxCents=3)`:吃到 `price > bestAsk + maxCents/100` 就停,返回 `fillableUsd`(带内能填多少)+ 真实 `capacityHit`。**优化器之前**就把"带内深度 < 最小注额($20)"的腿丢掉。配一个实盘 0:3 单测:断言 $20 被拒。

### 🔴 2 — 分区决策(more-markets 不是分区)
v1 **只用胜平负 3 格分区**(干净、深、ΣYES≈1.0、零新分区代码、零假墙暴露)。精确比分**仅作单腿加点**,且仅当该格近触可填 ≥ $20 时提供(favorite 格行,尾部自动回退到 result + `FELL_BACK_TO_RESULT` 文案)。让分/大小球(需要以 17 格比分为骨架做成员映射)**推迟**。

### 🔴 3 — 方案模式的裁决基线(decideStrategy 是"对冲"裁决,不是"方案"裁决)
`decide.ts:184` 的 `riskAfter.maxLoss > riskBefore.maxLoss → NO_GO` 会对**每个激进方案**(按设计就是抬高最坏损失)直接判 NO_GO。方案模式要:`riskBefore = 不下注基线(全 0)`,并**去掉/反转**这条最坏损失上升的 NO_GO 守卫。这是对"唯一裁决"的真实改动,要单测"激进方案不被自动 NO_GO"。

### 🔴 4 — 解析器(§2,完全没建):`discoverFixtures` + 队码图 + `parseBetIntent` + `resolveBet` + 两阶段现实闸。单测:"Spain vs Portugal"→not_found+真实建议;"England vs Croatia"→fixture。

## 5. 收益/风险呈现 + 诚实护栏(防劝赌)
方案页必须有:**情景赔付表**(结果→你的赔付→市场隐含去 vig 概率 `q`)、盈利概率 P(profit)、**诚实为负的期望值**、最大盈/亏、2–4 条自动生成的风险点。护栏(复查要求):
- **payoff 永远和 P(profit) 成对出现**,绝不单独亮一个大赔付数字。
- **最可能(亏损)那一行排在赔付表最上面**(按 q 降序——这是最有效的一条诚实设计)。
- 当 `pLoss ≥ 0.8` 或单腿 ≥90% 预算 → 渲染一条**等视觉权重**的硬提示"**约 88% 概率你会输掉全部 $B**"。
- `q` 标注为"market-implied (de-vigged)",**绝不**暗示是"我们的概率估计"或"你的 edge";相关性闭式只用于**解释文案**,不进定量。
- 这些字符串接进 `facts`,让现有的 `passesNumberGuardrail`(LLM 数字护栏)覆盖。

## 6. 头号风险
**假墙陷阱(§4.1)× 激进端(§3/§5)= 旗舰"全力表达"模式会按构造把真钱填进薄尾格、3–12 倍触价、还报"成交正常"。** 这是最差财务结果 + 最严重诚实违背 + 直接背叛产品定位。不是呈现层小修,需要新的带帽走簿 + 优化器前的硬最小填充闸 + 重写激进目标使它不去找假墙便宜腿。其余都在它下游。

## 7. v1 范围(批准的就是这个)
**下注类型:** 只 **胜平负(1X2)**(唯一干净深分区)。**精确比分仅作单腿加点**,逐格近触深度 ≥ $20 才给,否则回退 result。让分/大小球/半场/props/角球**全推迟**。

**滑块范围:** 只放 **s ∈ [0.4, 1.0]("平衡"↔"保护")**。保守端 = 已验证的补集对冲(今天的产品);中段在干净 result 分区上插值"最小方差↔表达"。**激进端 s<0.4("全力表达")锁住**,等 §4.1 带帽走簿 + §3 重写 U(最大赔付、深度闸控)+ "可能输光"硬提示都到位再开。

**开工前必须先有(缺一不可):**
1. 带帽近触走簿 + 实盘 0:3 拒 $20 单测(挡住一切)。
2. `discoverFixtures`+队码图+`parseBetIntent`+`resolveBet`+两阶段现实闸 + 单测。
3. 方案模式裁决基线(不下注基线;去掉最坏损失 NO_GO)+ 单测。
4. 重写 U(最大赔付 / 1/q 加权,深度闸控)+ 两端单测(λ→∞ 退化成对冲;λ→0 集中到**最高赔付**观点格,不是最高概率格)。
5. λ 按实盘 `Var_max` 归一化(去 κ)+ 单测(0.13 与 0.57 价格腿在 s=0.5 同姿态)。
6. 诚实护栏字符串(q 标注、强制负 EV 行、payoff+P(profit) 成对、pLoss≥0.8 的"输光"提示)接进 `facts`。
