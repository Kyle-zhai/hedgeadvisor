# HedgeAdvisor — 执行功能合规边界

**状态:** 动手前的边界文件。**写任何执行代码之前,和律师过一遍。**
**作者角色:** 懂监管的产品负责人 —— **不是律师。** 标 ⚖️ 的每一处都需要持牌的美国金融科技/CFTC 律师出具书面意见后才能上线。
**日期:** 2026-06-15 **范围:** 可选的一键执行层(L1 深链、L2 非托管中继)。L3 按决定不在范围内。

> **一句话论点:** 只要我们 (a) 永不碰密钥或资金、(b) 把美国用户挡在 offshore 场外、(c) 带上正确的免责声明,就能作为**非托管软件界面**现在上 L1、随后上 L2。两条硬线:**托管**(别越)和**美国路由**(别帮美国人上 offshore Polymarket)。

> **本文已根据对抗式事实核查修订(2026-06-15):** ① "L1 预填"已全部改为"深链到市场页(无订单预填)"——Polymarket 经核验**不支持** URL 预填订单,把"pre-fill"写进合规文件会误导法律评估。② §1.5 的措辞已放宽:核验确认"即使有 L2 凭据,创建用户订单仍必须用户重新签名",所以**凭据单独无法移动资金**,托管风险低于初稿暗示。③ 默认中继架构是**服务器瘦中继(架构 B)**,因为 CLOB 屏蔽浏览器跨域——服务器只过境转发已签订单、不持久化 secret,仍属非托管。

## 0. 两条硬线,先摆在最前面

一切都归结为两条规则。设计选择若在两条的安全侧,剩下的是声明卫生;若越过任一条,监管面跳一个数量级。

| # | 这条线 | 安全侧 | 越过触发 |
|---|---|---|---|
| **A. 托管** | 我们能否在任一瞬间,**在用户没有用自己的私钥签那笔具体交易的情况下**移动用户资金? | 否 —— 用户签每一笔;我们从不持有密钥或资金池 | 货币传输商/MSB 注册、~50 州 MTL、托管责任、黑客头号目标 |
| **B. 美国路由** | 我们是否在帮美国人接入 **offshore(global)** Polymarket? | 否 —— 美国用户只给建议 + 深链(L0/L1);永不 L2-on-global | 协助未注册 DCM(2022 年那次违规)、未注册中介敞口 |

CTA(对个性化建议收费)是**第三条独立轴**,**执行不改变它** —— 见 §2。

## 1. 托管线（规则 A）

### 1.1 什么算托管
托管 = **在没有用户逐笔签名的情况下移动用户资金的技术能力。** 满足以下任一即托管:
1. **你持有私钥**(或助记词,或任何能签名的 key share)。
2. **你持有汇集/混同资金** —— 用户的钱进了你控制的地址/账户,哪怕只是临时"为了路由"。
3. **你能在没有用户新签名的情况下发起转账** —— 例如给你自己控制的合约一个无限授权、有花费权的 session key、能不重签就提交的中继、或按常设授权行动的自动 bot。

以上都不成立 = **非托管**:用户用只有自己掌握的密钥签每一笔状态变更交易,你只**构造**订单 payload、**中继**已签消息。这种姿态下你是**软件界面**,不是持有价值的金融中介。

### 1.2 我们设计要满足的测试
> **HedgeAdvisor 在任何时刻都不得能够在用户没有用我们既不持有、也无法重建的密钥签那笔确切交易的情况下移动用户价值。**

这是 **FinCEN 2019 CVC 指引(FIN-2019-G001)** 的操作线:**非托管钱包软件提供者不是货币传输商**,因为它只提供软件;受监管的是**接收并再传输**价值的一方。"软件 vs 服务"的区分是全部关键。

### 1.3 越线 = MSB
若越入托管,联邦法把我们当 **MSB/货币传输商** → FinCEN 注册 + BSA/AML 程序 + SAR 报告,**外加** ~50 个司法辖区的**州级货币传输牌照(MTL)**(多年、数百万美元),**外加**成为混同加密资产的托管方 = 头号黑客目标。这也是 L3 被禁的同一类。**我们不去。**

### 1.4 保持非托管的 L2 设计约束（非可选）
- 用户连**自己的**钱包(WalletConnect/注入式)。我们永不见密钥。
- 我们构造**类型化订单**(EIP-712 CLOB 订单结构)交给钱包签。**用户的钱包签。**
- 我们中继**已签** payload。我们持有已签消息几毫秒;从不持有资金。
- **不给我们控制的合约任何授权。** 代币授权批给 **Polymarket 自己的** Exchange/CTF 合约(和用户直接交易时批的同一批),按协议允许的范围最小化 —— 永不批给 HedgeAdvisor 地址。
- **无 session key / 无常设花费权 / 无自动执行。** 每次对冲 = 一个离散的、用户签名的动作。⚖️ COUNSEL:确认"构造+中继用户已签 CLOB 订单"在 FinCEN 2019 下确属非托管。

### 1.5 CLOB 凭据这点（别跳过,已据核验放宽）
Polymarket 下单需要**从钱包签名派生的 API 凭据**外加签订单。**核验确认:文档原文"即使有 L2 头,创建用户订单仍必须让用户签订单 payload"——所以凭据单独无法动用户资金。** 因此持有 per-user 凭据**本身的托管风险低**。设计仍要求:凭据**客户端从用户自己的签名派生、只用于该用户自己的订单**,我们后端**绝不**持有能在没有新签名的情况下对用户余额下单的凭据。⚖️ COUNSEL 把这当一个确认项(而非高危项)过一下即可。

## 2. 执行不改变 CTA 问题

创始人最常见的误解,单列一节。

- **CTA 触发的是"建议",不是"执行"。** Commodity Trading Advisor 注册针对的是**对商品权益收费提供个性化、按持仓定制的建议**。我们是否**同时**执行,与我们是否是 CTA 无关。
- **相关豁免是 CFTC Rule 4.14(a)(9)**,只豁免**标准化/非个性化**建议(简报、网站、非定制软件),且**不针对个别客户的持仓/情况定制**、**不指挥客户账户**。核验确认:即便把同样的建议给"情况类似的客户",只要它源自客户特定持仓信息,豁免照样**丧失**。
- **我们落在哪:** 从**用户这个具体持仓**算出的对冲建议,表面就是**定制的、按持仓的建议** —— 4.14(a)(9) **不**豁免这类。若我们**收费**,CTA 敞口**与执行无关地**已经存在。⚖️ COUNSEL:这是**整个产品**最大的常设问题。可选项:把建议做成非个性化/标准化(市场级而非账户定制)、不对个性化层收费、或注册。**别让"我们只是加个按钮"盖住这个核心产品里早就存在的风险。**

**执行额外引入的是:中介/未注册中介敞口。** 若我们把美国用户路由进商品权益交易,我们开始像在为交易所**招揽或接受订单** —— 这触及美国衍生品世界的 **Introducing Broker / FCM** 注册与**中介化**模型(§3)。这与托管无关,是另一根针。

> **净:执行不动 CTA 针,它加一根中介针。当两个独立风险管理。**

## 3. 美国 vs 非美路由（规则 B）—— 路由矩阵

### 3.1 为什么是第二条硬线
**global(offshore)Polymarket 屏蔽美国交易。** 帮美国人接入 offshore 正是 **CFTC 2022 年罚的行为**:Blockratize/Polymarket 付 **$1.4M** 并被勒令停止,因其经营**未注册/未指定**的、对美国人开放的事件型二元期权设施。若**我们的软件**成了把美国用户送上 offshore 盘的入口,我们就在协助同一违规。**所以:美国用户必须被挡在任何 L2-on-global 路径外。**

### 3.2 唯一合规的美国执行路径
对美国用户,唯一合法执行场所是 **Polymarket US 的中介化模型**:一个 **CFTC 监管的 DCM**(经 **QCEX** 收购,~$112M,amended Order of Designation 于 **2025-11-25** 获批),美国用户**通过持牌 FCM/经纪商、全 KYC、美元结算**交易 —— **不是**连加密钱包点对点。该模型**更重**(申请制 API、FCM 关系、KYC 透传、美国衍生品规则),**不是 MVP 路径**。**建议:MVP 里美国用户只给 L0/L1** —— 建议 + 深链到相应的 Polymarket US 界面,无在场执行。

### 3.3 路由矩阵（辖区 × 执行级别）

| 辖区 | L0 仅建议 | L1 深链到市场页(无预填) | L2 非托管在场中继 | L3 托管/bot |
|---|---|---|---|---|
| **美国人** | ✅ 允许(受 CTA 约束,§2) | ✅ 只深链到 **Polymarket US(中介/DCM)** 界面 —— **绝不**到 offshore 盘 | ⛔ **MVP 屏蔽。** 仅可能在后期经 Polymarket US 中介 API(FCM/KYC/申请制)实现 —— 重、后置、律师 gated | ⛔ 禁止 |
| **非美、场所合规** | ✅ 允许 | ✅ 深链到 global Polymarket | ✅ **L2 快速跟进目标** —— 非托管中继到 global CLOB,用户签 | ⛔ 禁止 |
| **受制裁/OFAC** | ⛔ 整体拦 | ⛔ 拦 | ⛔ 拦 | ⛔ 禁止 |

### 3.4 地理拦截是真控制,不是打勾
- 至少 **IP 地理定位 + 辖区声明**;⚖️ COUNSEL 决定是否要处理 VPN 规避和更强地理围栏(2022 那道令意味着"我们尽力了"必须可信,不能装样子)。
- **钱包/区域信号**:连接钱包或会话定位为美国 → **L2 路径不渲染**,fail-closed 退回 L1。
- 待记录的决定:MVP 里美国用户给不给 L1-到-Polymarket-US。默认建议:**给 L1 到 Polymarket US 界面,L2 对美国禁用,且美国用户的 L1 绝不深链到 offshore。**

## 4. L1 / L2 / L3 各触发什么

| 维度 | **L1 深链到市场页(无预填)** | **L2 非托管中继** | **L3 托管/资金池/bot** |
|---|---|---|---|
| 我们做什么 | 打开 Polymarket 精确市场页;**用户在 Polymarket 上确认** | 构造 EIP-712 订单;**用户在自己钱包签**;我们中继已签消息到 CLOB | 持密钥/资金;代用户提交 |
| 碰资金/密钥? | **从不** | **从不**(用户签,我们中继) | **是** |
| 托管(规则 A) | 无 | **若** §1.4/§1.5 成立则无 | **越线** → MSB |
| MSB/MTL | 否 | 否(非托管软件,FinCEN 2019) | **是** —— FinCEN + 50 州 MTL |
| 中介/IB 问题 | 极小 | **存在** —— 构造+中继更像招揽/中介;按场所与辖区 gated | 严重 |
| CTA 轴(§2) | L1 不改变 | L2 不改变 | 不改变 |
| 美国-on-global 地理拦 | 建议(美国别深链到 offshore) | **必须** | N/A(已禁) |
| 黑客目标责任 | 无 | 无(不持资金) | **高** |
| 建设成本 | 几天 | 数周 + 律师 | N/A —— 禁止 |
| 结论 | **MVP 上** | **快速跟进,非美,律师审** | **永不** |

> **竞品注:** PredictEngine、TurbineFi 已在 Polymarket 上做**非托管**自动执行(TurbineFi:"不接触你的资金";PredictEngine:非托管、no-KYC、~30s 钱包)。这**验证了 L2 姿态可行** —— 但他们的 no-KYC + 自动执行应被假定带着**我们明确选择去管理的美国路由和中介风险**,不是我们跳过地理拦的许可证。⚖️ COUNSEL:别把竞品行为当安全港。

## 5. 免责声明

### 5.1 必备用户面文案（显著、需确认,不能埋在 ToS)
1. **非财务/投资建议。** "HedgeAdvisor 提供信息工具与建议。这不是财务、投资、法律或税务建议。你对自己的交易决定负全责。"
2. **亏损风险。** "预测市场与衍生品交易涉及重大风险,包括本金全损。"
3. **非托管。** "你始终掌控自己的密钥和资金。HedgeAdvisor 从不持有、转移或接触你的资金或私钥。每笔交易都由你在自己的钱包签名。"
4. **不保证成交/价格变动。** "订单簿很薄,价格会动。你看到的价格是估计;实际成交价和是否成交都不保证。我们在你点击的瞬间重新报价。"
5. **内盘对冲的负 EV 本质。** "在同一订单簿内对冲通常在期望上是亏的(你穿价差并双边付费)。对冲降低方差/下行,不预期盈利。" (诚实给数:体育 taker 费 p=0.5 处峰值约 0.75%;maker ~0%;卖单免费;兑付 $1.00、0 费。)
6. **辖区。** "仅在允许地区可用。在场执行不向美国人开放。继续即确认你有资格在所选场所交易。"
7. **第三方场所。** "执行发生在第三方 Polymarket 上。我们是独立界面,与 Polymarket 无关联、未获其背书。"

### 5.2 出现位置
- **执行前确认屏(强制)**:**重报价**("你将为 N 份、M 笔订单支付 ~$X,预估费 $Y,最差价 $Z")+ #3(非托管)+ #4(不保证成交/价格变动)+ #5(负 EV),在**动作那一刻**显示,签名前需明确"我已理解"。
- **首次运行/引导**:全套,确认一次。 **页脚/ToS**:全套,持续。
- **L1 专门**:讲清用户正在**离开**到 Polymarket 并**在那边确认**,无歧义地表明我们不提交。

⚖️ COUNSEL 敲定确切措辞(尤其 #1、#6)并确认确认屏流程满足披露充分性。

## 6. 应用商店 / 支付通道风险 → web 优先
- **苹果/谷歌政策风险。** 一个 facilitating 衍生品/赌博相邻交易的非托管 dapp 常踩 App Store/Play Store 政策(实金游戏门、地理限制、加密交易规则、IAP 纠缠)。**建议:web 优先(PWA)**;原生 app 须经律师 + 平台政策评审后再议。
- **支付通道敏感性。** 卡处理商(Stripe 等)对赌博相邻 MCC 敏感。我们的**订阅/建议**收入(工具 SaaS)应与任何看起来像处理交易/赌博资金的东西**干净分离**。因为我们**非托管**,**从不**处理交易资金 —— 保持如此,并把**计费描述为软件/SaaS**,不是市场接入。⚖️ COUNSEL/财务:确认处理商条款;引导时如实披露商业模式以免账户冻结。

## 7. ToS / 数据 / 制裁
- **商用条款张力。** Polymarket ToS 授予"个人、非商用"许可。一个**程序化中继订单**的付费产品很可能算商用 → **常设 ToS 风险**。缓解:Polymarket 对 builder 友好(2026/2 收购 **Dome**),读接口公开。但**下单/撤单需 API 凭据 + 钱包签名**,商用规模的程序化中继正是 ToS 摩擦点。⚖️ COUNSEL + BD:**建 L2 前**向 Polymarket 取得**书面确认/builder 协议/API 条款**,允许第三方界面做用户已签订单的程序化中继。**别把"builder 友好"当成"合同上获准"。**
- **OFAC/制裁筛查 —— 非托管也适用。** 非托管**不**豁免制裁法。为受制裁人/辖区 facilitating 交易可能违法,无论谁持资金。落实:**OFAC 辖区地理拦** + ⚖️ COUNSEL 关于**钱包地址筛查**(黑名单),与 §3.4 的美国地理拦同一道闸,fail-closed。
- **数据/隐私:** 若存用户持仓/钱包地址来算对冲,适用标准隐私义务;最小化留存;不存任何放大泄露影响面的东西(Neon/Upstash 静态加密,永不存密钥材料)。

- [ ] **计费 = 软件/SaaS**,与任何资金流(本就没有)干净分离。(§6)

### 8.3 上线顺序
1. **L1(深链)= MVP,所有合规辖区**(美国→Polymarket US;非美→global)。表面最小。§8.2 披露 + 地理逻辑落地即可上;§8.1 中针对 L1 的项较轻。
2. **L2(非托管中继)= 快速跟进,仅非美**,gated on §8.1 的托管意见、ToS/builder 条款、中介分析、制裁程序。
3. **L3 = 永不。**

### 见律师前先读的诚实不确定性
- **非托管→非 MSB** 的结论有 FinCEN 2019 强支撑,但**事实相关**;**API 凭据机制**是可能动摇之处 —— 代码评审 + 律师都要钉死。
- **CTA 敞口是最被低估的风险,且今天就在核心产品里、与执行无关地存在。** 别让执行话题盖过它。
- **州级**货币传输与**非美本地**监管各异;本文以美国联邦为中心。L2 的每个目标非美辖区需单独核。
- 竞品做非托管/no-KYC 自动执行是**模型可行的证据,不是合规的证据。** 我们在美国路由和披露上故意更保守。
